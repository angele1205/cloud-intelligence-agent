Architecture Overview

CostOp Intelligence Agent is a real-time FinOps monitoring and investigation agent built on Amazon Bedrock AgentCore. It autonomously detects cost anomalies, investigates root causes using hypothesis-driven reasoning, and delivers structured reports via email and Slack.

The system extends the AWS FinOps Agent pattern with proactive alerting, pattern memory, per-agent cost tracking, and a web-based investigation console.

Well-Architected Decision: The agent uses Amazon Bedrock AgentCore (managed container runtime) instead of self-managed ECS/EKS to minimize operational overhead while maintaining full tool-use capabilities via the Strands SDK.

AWS Services Used

Service Purpose Configuration Documentation
Amazon Bedrock AgentCore Managed container runtime for the AI agent. Handles scaling, auth, and Gateway integration. Container image from ECR Private; PUBLIC network mode; IAM role with 14 policy statements Docs
Amazon Bedrock Foundation model inference (Claude Sonnet 4.6 / 4.5 / Haiku 4.5). Powers reasoning and tool use. Configurable model ID; max 12,000 tokens; Converse API with streaming Docs
Amazon Bedrock AgentCore Memory Persistent conversation memory across sessions. Stores interaction history per user/session. Configurable retention (7–365 days); per-session and per-user isolation Docs
Amazon Cognito User authentication (User Pool) and AWS credential vending (Identity Pool) for the web UI. Admin-only user creation; SRP + password auth; strong password policy; email verification Docs
Amazon CloudWatch Real-time Bedrock metrics monitoring. 5 alarms: Token Spike, RPM, TPM Quota, Throttles, Errors. 1–5 min periods; Sum/Maximum stats; configurable thresholds via parameters Docs
Amazon CloudWatch Logs Bedrock model invocation logging. Enables per-caller token analysis and loop detection. Log group: /aws/bedrock/modelinvocations; Insights queries for aggregation Docs
AWS CloudTrail API activity audit trail. Used to identify WHO made changes and WHEN during investigations. LookupEvents API; filters by EventSource; 6-hour default lookback Docs
AWS Cost Explorer Cost and usage data. Provides daily spend by service, anomaly detection, and forecasting. BlendedCost metric; SERVICE grouping; 7-day default lookback; anomaly detection Docs
Amazon DynamoDB Three tables: Patterns (learned incidents), Investigations (results), Topology (resource graph). PAY_PER_REQUEST billing; simple key schema (pattern_id, investigation_id, resource_id) Docs
Amazon EventBridge Event routing for proactive investigations. Captures alarm state changes, budget alerts, and cost anomalies. 3 rules: alarm-trigger, budget-trigger, anomaly-trigger → Bridge Lambda Docs
AWS Lambda Two functions: Bridge Lambda (event→agent invocation) and Build Trigger (ECR image copy). Python 3.12; 120s/600s timeout; 256MB memory; 5-min dedup cooldown Docs
Amazon SNS Alert delivery. Sends investigation results and alarm notifications via email. Email protocol subscription; auto-created or user-provided topic ARN Docs
AWS Budgets Monthly cost guardrail for Bedrock spend. Alerts at 80% threshold via SNS. COST type; MONTHLY; filtered to Amazon Bedrock service; configurable limit Docs
AWS Cost Anomaly Detection ML-based anomaly detection for unexpected cost spikes. Triggers EventBridge events. Conditional creation; integrates with EventBridge for automated investigation Docs
Amazon ECR Private container registry for the agent Docker image. Lifecycle policy keeps 3 images max. MUTABLE tags; lifecycle: keep 3 images; image copied from public ECR via CodeBuild Docs
AWS CodeBuild Copies the public agent image to the private ECR repo during stack creation. ARM container; BUILD_GENERAL1_SMALL; 10-min timeout; NO_SOURCE type Docs
AWS Amplify Hosting Static web hosting for the investigation console SPA. Auto-deployed during stack creation. Single-page app; main branch; config injected at deploy time via Custom Resource Docs
AWS IAM Least-privilege roles for runtime, Lambda, CodeBuild, Cognito authenticated users, and memory. 6 IAM roles; scoped policies per resource; STS AssumeRole for cross-service access Docs
AWS Service Quotas Runtime quota checks for Bedrock TPM/RPM limits. Enables proactive quota increase requests. ListServiceQuotas + GetServiceQuota + RequestServiceQuotaIncrease Docs

AWS Well-Architected Framework

Assessment against the six pillars of the AWS Well-Architected Framework:

Operational Excellence

Strengths: Single-stack IaC deployment; auto-investigation on alarms; structured investigation output; pattern memory for continuous improvement; conversation history via AgentCore Memory.

Opportunities: Add CloudWatch dashboard for agent health; implement canary testing for agent responsiveness.

Security

Strengths: Cognito-based auth with strong password policy; SigV4 signing on all API calls; least-privilege IAM roles (6 separate roles); admin-only user creation; non-root container user; destructive actions require confirmation.

Opportunities: Add MFA; implement API request throttling; enable AWS WAF on Amplify endpoint.

Reliability

Strengths: Managed services (AgentCore, DynamoDB, Lambda) with built-in HA; event-driven architecture with dedup prevents storm amplification; conditional resource creation avoids conflicts with existing infrastructure.

Opportunities: Add DLQ to Bridge Lambda; implement circuit breaker on agent invocations; add retry logic with exponential backoff.

Performance Efficiency

Strengths: Skill-based routing reduces token usage by selecting targeted prompts; 12 combined tools (from 36) reduce round-trips; ARM-based CodeBuild for efficient image builds; configurable model selection (Haiku for speed, Sonnet for quality).

Opportunities: Implement response caching for repeated queries; add streaming responses for web UI.

Cost Optimization

Strengths: PAY_PER_REQUEST DynamoDB; Lambda free tier; 5-min dedup cooldown prevents cascading costs; budget alerts with configurable threshold; model flexibility (Haiku = 8x cheaper); ECR lifecycle keeps only 3 images.

Opportunities: Add per-session token budgets; implement graduated response (Haiku first, escalate to Sonnet for complex cases).

Sustainability

Strengths: Serverless/managed services minimize idle compute; ARM-based builds; on-demand scaling with no over-provisioning; skill routing minimizes unnecessary LLM processing.

Opportunities: Track and report carbon footprint per investigation; implement response caching to reduce redundant model calls.
